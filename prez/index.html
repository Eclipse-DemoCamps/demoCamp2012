Eclipse 4: Application model

  • Structure of your app described by a model (i.e. Parts*)
  • But not the content of the Part (e.g. SWT widgets)
  • Arrange Parts using Perspectives, Stacks and Sashes
  • Connect the UI Part through the Class URI
*Part: View and/or Editor

Source: http://www.flickr.com/photos/29714836@N08/2995851157/ by curiouslypersistent

Eclipse 4: Event broker

  • Publish/Subscribe pattern, with Topics (similar to MQTT)
  • Subscribe
@Inject
@Optional
void handleMessage(@UIEventTopic("demoCamp/nantes") String string) {
	// do something
}

@Inject
@Optional
void handleMessage(@UIEventTopic("demoCamp/nantes") MyObject myObj) {
	// do something else
}

Eclipse 4: Event broker

  • Publish
@Inject
@Optional
private IEventBroker eventBroker;
@Inject
public MyPart(Composite parent) {
	...
	MyObject myObj = new MyObject("Hello demo camp!")
	Map<String, Object> map = new HashMap<String, Object>();
	map.put("publishing a string", "Hello demo camp!");
	map.put("publishing an object", myObj);
	// async
	eventBroker.post("demoCamp/nantes", map);
	// sync
	eventBroker.send("demoCamp/nantes", map);
	...
}

Eclipse 4: CSS Styling

	id="product"
	point="org.eclipse.core.runtime.products">

        name="demoCampNantes"
        application="org.eclipse.e4.ui.workbench.swt.E4Application">
		...
        <property
		    name="applicationCSS"
		    value="platform:/plugin/demoCampNantes/css/default.css">
        </property>
    </product>
</extension>
